# UZ News Bot

Автоматический сбор новостей из RSS → фильтрация → Telegram.

---

## Кнопки (файлы для запуска)

### `collect.bat`
**Что делает:** Собирает свежие новости из RSS прямо сейчас + отправляет их в Telegram.

Последовательность:
1. Идёт на kun.uz, upl.uz, gazeta.uz — берёт новые статьи
2. Проверяет каждую по ключевым словам (фильтр)
3. Сохраняет подходящие в базу данных (дубли пропускает)
4. Отправляет в Telegram всё что ещё не было отправлено

**Когда использовать:** Хочешь получить свежие новости прямо сейчас, не ждать расписания.

---

### `send_now.bat`
**Что делает:** Отправляет в Telegram только то что уже есть в базе данных, но ещё не было отправлено.

Ничего не собирает с интернета. Просто берёт накопленное из БД.

**Когда использовать:** Если collect уже запускался, но Telegram не получил сообщение (ошибка сети, бот был выключен и т.д.).

---

### `setup_scheduler.bat`
**Что делает:** Настраивает автоматический запуск на 12:00 и 22:00 каждый день.

Запускать **один раз**, от имени Администратора (ПКМ → "Запуск от имени администратора").

После настройки — каждый день в 12:00 и 22:00 бот сам соберёт новости и отправит в Telegram. Компьютер должен быть включён.

---

### `setup.bat`
**Что делает:** Проверяет что всё настроено правильно — Python, библиотеки, ключи API, папки, база данных.

Запускать если что-то перестало работать — покажет где проблема.

---

## Как это работает вместе

```
Авто (12:00 и 22:00):
  RSS → фильтр → БД → Telegram

Ты вручную:
  collect.bat → Telegram (сразу)
  send_now.bat → Telegram (из БД)
```

Логика отправки:
- Каждая новость в БД имеет флаг `sent_tg = 0` (не отправлена) или `1` (отправлена)
- После отправки флаг меняется на `1` — повторно не отправится никогда
- `send_now.bat` берёт только `sent_tg = 0`
- Поэтому одна и та же новость не придёт дважды

## Формат сообщения в Telegram

```
🗞 НОВОСТИ — 03.04 12:00

1. 03.04 11:30  Заголовок новости — кликабельная ссылка
2. 03.04 10:15  Другая новость
3. 02.04 22:40  Вчерашняя новость
...

💡 Копируй нужные → вставь в Claude.ai
```

Время показывается по Ташкенту (UTC+5).

## Файлы проекта

```
uz-news-bot/
├── collect.bat          — Собрать + отправить в TG прямо сейчас
├── send_now.bat         — Отправить накопленное в TG
├── setup_scheduler.bat  — Настроить авто-запуск (12:00 / 22:00)
├── setup.bat            — Проверить настройки
│
├── config/
│   ├── config.yaml      — Источники RSS, технические настройки
│   └── preferences.yaml — Правила фильтрации, стиль TTS
│
├── src/
│   ├── auto_collect.py  — Основной скрипт: сбор + фильтр + отправка
│   └── send_pending.py  — Только отправка из БД
│
├── data/
│   ├── news.db          — База данных (все новости, флаги отправки)
│   └── auto_collect.log — Лог всех запусков
│
└── .env                 — Ключи API (не редактировать руками)
```

## Добавить / убрать источник RSS

Открыть `config/config.yaml`:

```yaml
sources:
  - name: "Kun.uz"
    url: "https://kun.uz/ru/news/rss"
    category: "general"

  # Добавить новый:
  - name: "Daryo.uz"
    url: "https://daryo.uz/rss"
    category: "general"
```

---

## Как найти RSS ссылку для сайта

Обычно это одно из:
- `сайт.uz/rss`
- `сайт.uz/rss.xml`
- `сайт.uz/feed`

Или поиск в Google: `"daryo.uz" RSS`

## Ключи API (.env)

```
TELEGRAM_BOT_TOKEN=   — токен бота от @BotFather
TELEGRAM_CHAT_ID=     — твой числовой ID (от @userinfobot)
ANTHROPIC_API_KEY=    — Claude API (если используешь AI фильтрацию)
ELEVENLABS_API_KEY=   — ElevenLabs (для аудио)
ELEVENLABS_VOICE_ID=  — ID голоса в ElevenLabs
```
